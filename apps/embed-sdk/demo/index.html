Afianco Embed SDK — Demo

Local dev playground. Run pnpm --filter @afianco/embed-sdk dev e visita questa pagina. Lit hot-reload attivo via Vite.

4. <afianco-storefront-init> — bootstrap (Step 22)

Root provider del widget Stream A. Chiama /api/public/embed/init/{slug} al mount, espone il context via Lit a tutti i child components.

Per testare con backend locale: assicurati che bottega-demo abbia store.allowed_origins con http://localhost:3001 (porta dev di Vite). Vedi snippet python in commit msg per setup.

4b. <afianco-product-card> standalone (.product property)

Card singolo iniettato come property — niente fetch. Usato dal product-grid futuro (Step 24) per renderizzare ogni elemento senza N+1 round-trip. Click "Aggiungi al carrello" emette evento afianco:add-to-cart.

5. Storefront-init senza slug (warning state)

6. <afianco-login> + <afianco-signup> (Step 27)

Customer auth standalone — utili per merchant che vogliono offrire pagine dedicate /accedi e /registrati. Token JWT salvato automaticamente in localStorage[afianco_token_<slug>].

7. <afianco-customer-portal> (Step 28 — Track C completion)

Area Personale auth-gated. Mostra profilo + storico ordini (read-only V1). Se il customer non e' loggato (no token in localStorage[afianco_token_<slug>]) mostra un prompt "Accedi" e dispatcha afianco:auth-required. Per testare il flow completo, fai login con <afianco-login> sopra, poi ricarica la pagina.

Attributi: initial-tab="profile|orders", show-logout, title.
