🧪 AFianco Embed — Local Dev Test Page

Testing scenario:
  • Bundle JS servito da: http://localhost:3000/embed/v1/afianco-embed.es.js
  • Backend API: http://localhost:8000/api/public/embed/*
  • Slug store da testare: cambia YOUR-STORE-SLUG sotto con il tuo
Prereq:
  1. Backend + frontend dev running
  2. Almeno 1 store pubblicato con slug valido
  3. Origin null (file:// protocol) OPPURE serve la pagina via: python3 -m http.server 8080 e poi accedi a http://localhost:8080/embed-local-test.html
  4. Aggiungi l'origin nella modale "Condividi" del store (es. http://localhost:8080 oppure null se apri file direttamente)

Embed widget

Se la pagina e' vuota:
  1. Apri DevTools (Cmd+Option+I) → Console — vedi errori?
  2. Network tab → afianco-embed.es.js caricato? /init/{slug} chiamato?
  3. Possibili problemi:
    • 404 init → slug sbagliato
    • 403 origin → aggiungi origin in allowed_origins via "Condividi" modale
    • CORS error → origin del browser non whitelistato
